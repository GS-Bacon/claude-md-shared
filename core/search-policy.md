# Search Policy (推測禁止・調査必須)

## 基本原則

- **推測による作業を禁止する**。分からないことは必ず調査してから答える
- 「たぶん」「おそらく」「〜のはず」で書き始める前に、調査ツールを呼べないか考える

## 調査手段

調査が必要な状況に応じて使い分ける:

| 状況 | 手段 |
|---|---|
| プロジェクト内のコード・設定の所在 | Glob / Grep / Read |
| 不明な大規模コードベースの探索 | Explore サブエージェント |
| 外部 API・ライブラリの最新仕様 | WebFetch / WebSearch |
| 「最新の」「現在の」「公式の」を含む話題 | WebSearch で裏取り必須 |
| バージョン依存の挙動 | 公式ドキュメントを参照 |

## やってはいけないこと

- 知らない API・ライブラリ・設定値を「たぶんこうだろう」で書く
- 仕様確認なしに「最新のバージョンでは〜」と書く
- ユーザーに「これで合ってますか?」と確認を投げる前に、自分で調べられる範囲は調べる

## ユーザーへの確認のタイミング

調査して尚不明な場合・複数の妥当な選択肢がある場合に限り、ユーザーに聞く。
聞くときは具体的な選択肢を提示する(「どうしますか?」ではなく「A/B のどちらにする?根拠は〜」)。
